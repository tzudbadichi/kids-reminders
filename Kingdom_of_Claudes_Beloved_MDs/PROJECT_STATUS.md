# PROJECT_STATUS - מצב הפרויקט, הכרעות ושלבים הבאים

מסמך זה הוא נקודת ההמשך לעבודה. סשן חדש שקורא אותו אמור להבין איפה הפרויקט עומד, אילו הכרעות התקבלו ולמה, ומה הצעד הבא - ולהמשיך מכאן.

## תמצית המוצר
אפליקציית ווב (PWA) להורים: נכנסים, בוחרים ילד מתוך רשימה (אפשר לנהל שמות), בוחרים תאריך מלוח שנה, ומדביקים הודעת ווצאפ שהתקבלה מהמורה או הגננת. אופציונלית שולחים את הטקסט ל-AI שמחלץ את מה שצריך להביא (למשל "חולצה לבנה", "חטיף ובקבוק מים"), והתוצאה חוזרת לעריכה ואישור. בשמירה זה נשמר לכל ילד וליום הנכון. בכל בוקר בשעה שהוגדרה (ברירת מחדל 6:30) קופצת התראה עם מה שצריך לאותו יום, מקובצת לפי ילד. המשתמשים מישראל בלבד.

## מצב נוכחי
- **הליבה נבנתה** (פאזה 1-2): התחברות, ניהול ילדים, הוספת תזכורת (הזנה ידנית כברירת מחדל + כפתור "חלץ עם AI" אופציונלי), ותצוגת "מה צריך להביא" לפי תאריך. הכל מאובטח ב-RLS.
- **התחברות בשם משתמש + סיסמה** (לא אימייל) - האפליקציה בונה email פנימי מהשם. עיצוב חודש למובייל-פירסט: ניווט תחתון, אווטארים צבעוניים, גופן Rubik. הסשן נשמר במכשיר (זכירת התחברות) ותומך בכמה מכשירים לאותו משתמש. פירוט ב-`FRONTEND.md`.
- **רץ מול Supabase חי**: `src/config.js` מקומי (ב-`.gitignore`) עם ה-URL וה-publishable key. הסכמה הורצה, "Confirm email" כבוי, וטלגרם פרוס ונבדק. שרת מקומי: `python -m http.server 5173` -> `http://localhost:5173/src/`.
- **הריפו**: https://github.com/tzudbadichi/kids-reminders (ציבורי, ענף main, חשבון tzudbadichi).
- **מיקום**: האפליקציה ב-`c:\Projects\my\kids-reminders`. תיקיית-העל `c:\Projects\my` מכילה פרויקטים אחרים - לא לאתחל git ולא לדחוף ברמת תיקיית-העל.

## הכרעות מרכזיות (החלטה / נימוק / מה נדחה)

1. **אחסון: מסד הנתונים של האפליקציה (Supabase Postgres) כמקור אמת.**
   נימוק: נתונים מובנים לכל ילד/תאריך מאפשרים התראה חכמה ומקובצת.
   נדחה: יומן גוגל בלבד (טקסט חופשי, קשה לפרסר, התראה גנרית).

2. **פלטפורמה: Supabase כהכל-באחד** (Postgres + Auth + RLS + Edge Functions); פרונטאנד סטטי ב-GitHub Pages. *לא* על שרת ה-IIS של מילגם.
   נימוק: אפליקציה רב-משתמשית מופצת צריכה התחברות והפרדת נתונים - Supabase נותן הכל בחינם.
   נדחה: Neon (רק DB, היה צריך לבנות auth/API לבד); GitHub Pages בלבד (סטטי, אין שרת להחביא מפתחות).

3. **AI: Google Gemini (free tier), אופציונלי, דרך Edge Function.**
   נימוק: free tier מסיר את שיקול העלות בהפצה רחבה. הקריאה עוברת בצד-שרת כדי להחביא את המפתח (אסור בדפדפן באפליקציה רב-משתמשית). החילוץ אופציונלי לגמרי: כפתור "חלץ עם AI", אבל ההזנה הידנית היא ברירת המחדל ואפשר לדלג על ה-AI.
   נדחה כברירת מחדל: Claude (אין free tier; מנוי 20$ הוא Claude Pro - לא גישת API). השכבה נשמרת ניתנת להחלפה.

4. **התראות: שני ערוצים - Web Push + בוט טלגרם.**
   Web Push: PWA מותקנת כקיצור במסך הבית (בלי חנות אפליקציות), חינמי; באייפון עובד רק אם מותקנת למסך הבית.
   טלגרם: עמוד שדרה אמין שעובד בכל מכשיר כולל אייפון.
   נדחה: WhatsApp Business API (עלות/אישור), אימייל (פחות בולט).

5. **תזמון: GitHub Actions cron (טריגר חיצוני יומי) שמפעיל Edge Function.**
   נימוק: שולח את ההתראות וגם שומר את פרויקט ה-Supabase ער (המסלול החינמי משהה פרויקט אחרי כ-7 ימי חוסר פעילות, וההוצאה מ-PAUSE ידנית).
   נדחה: pg_cron פנימי (לא בטוח שנחשב "פעילות" שמונעת PAUSE).

6. **אזור זמן: ישראל בלבד - Asia/Jerusalem קבוע, בלי בחירה אישית.**
   נימוק: כל המשתמשים מישראל. עדיין מחשבים לפי Asia/Jerusalem (לא UTC) כדי ששעון קיץ/חורף לא יזיז את ה-6:30.

7. **התחברות: שם משתמש + סיסמה (לא אימייל).**
   נימוק: חוויה פשוטה להורים, בלי אישור מייל. Supabase דורש email כמזהה, ולכן בונים email פנימי `<username>@kids-reminders.app` שהמשתמש לא רואה. דורש "Confirm email" כבוי. שם המשתמש מוגבל ל-ASCII כי החלק שלפני ה-`@` לא יכול להכיל עברית.
   נדחה: התחברות באימייל (דורש אישור מייל, מסורבל), עברית בשם המשתמש (בעיית פורמט email).

8. **לא לקשר את Supabase ל-GitHub.** דיפלוי Edge Functions דרך Supabase CLI.
   נימוק: Branching בתשלום והאינטגרציה מניחה workflow של migrations שאין לנו (schema.sql יחיד). ה-cron של פאזה 5 הוא GitHub Action שקורא ל-Edge Function ב-HTTP - נפרד ובסדר.

## מבנה הנתונים
פירוט מלא ב-`DATABASE.md`. בקצרה: profiles (כולל notification_time, telegram_chat_id), children, reminders (items כ-jsonb, source_text), push_subscriptions, notification_log. RLS לכל טבלה (auth.uid()), וטריגר שיוצר פרופיל בהרשמה.

## שלבי בנייה
- פאזה 1-2 (ליבה): **בוצע** מקומית - צריך הרצה ראשונה מול Supabase חי.
- פאזה 4 (טלגרם): **בוצע** - הבוט `KidsReminderBot` נוצר, שתי הפונקציות פרוסות בדאשבורד (`telegram-webhook` ציבורית עם אימות secret, `send-telegram`), ה-webhook רשום, והחיבור + "שלח התראת בדיקה" נבדקו בהצלחה. פירוט ב-`TELEGRAM.md`.
- פאזה 5 (תזמון בוקר): **ממתין** - צריך פונקציית שליחה אצווה (לולאה על כל המשתמשים לפי `notification_time`) + GitHub Actions cron יומי שמפעיל אותה. כרגע קיימת רק שליחה ידנית למשתמש המחובר.
- פאזה 6 (פריסה ל-GitHub Pages): **ממתין** - נדרש כדי שאפשר יהיה להשתמש מהטלפון (כרגע רץ רק מקומית על המחשב).
- פאזה 3 (חילוץ AI): Edge Function `extract-items` עם מפתח Gemini - **ממתין** (אופציונלי; ההזנה הידנית עובדת).
- פאזה 4ב (Web Push): **ממתין** (אופציונלי; טלגרם כבר מכסה התראות).
- ליטוש PWA (אייקונים): **ממתין**.

## הצעד הבא המיידי
הליבה, ההתחברות בשם משתמש, והתראות טלגרם (חיבור + שליחה ידנית) עובדים מול Supabase חי. נותר, לפי עדיפות:
1. **תזמון בוקר (פאזה 5)** - הלב של המוצר: פונקציית שליחה אצווה + GitHub Actions cron יומי ל-6:30 (גם שומר את פרויקט ה-Supabase ער). בלי זה אין שליחה אוטומטית.
2. **פריסה ל-GitHub Pages (פאזה 6)** - כדי להשתמש מהטלפון (כרגע רק מקומית).
3. **AI עם Gemini (פאזה 3)** - אופציונלי, נוחות בלבד.
4. **Web Push (פאזה 4ב)** - אופציונלי, טלגרם כבר מכסה.
5. **אייקוני PWA** - ליטוש.
