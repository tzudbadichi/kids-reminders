# SCHEDULING - תזמון התראות הבוקר

שליחה אוטומטית של תזכורות הבוקר בטלגרם, מופעלת מבחוץ ב-cron.

## רכיבים
- `supabase/functions/send-morning/index.ts` - פונקציית שליחה אצווה. בכל הרצה: מוצאת משתמשים שמחוברים לטלגרם, ששעת ההתראה שלהם כבר עברה היום (שעון ישראל), שיש להם פריטים להיום, ושטרם נשלחה אליהם הודעה היום, ושולחת. מאומתת ב-`x-cron-secret` מול הסוד `CRON_SECRET`.
- `.github/workflows/morning-notify.yml` - GitHub Actions cron שרץ כל 15 דקות וקורא ל-`send-morning` עם ה-`CRON_SECRET` (מ-GitHub Secret).

## למה "כל 15 דקות" + לוגיקת catch-up
GitHub Actions cron לא מדויק לדקה ועלול להתעכב או לדלג. לכן הפונקציה לא בודקת "האם השעה היא בדיוק X" אלא **"מי שעבר את שעת ההתראה שלו היום ועוד לא נשלח לו"** - כך כל הרצה משלימה את מה שפוספס. ריצה כל 15 דקות גם שומרת את פרויקט ה-Supabase ער (מונע השהיה במסלול החינמי).

## מניעת כפילויות
טבלת `notification_log` עם `unique(user_id, sent_on, channel)`. אחרי שליחה מוצלחת נרשמת שורה ל-`channel='telegram'` עם תאריך היום; הרצות נוספות באותו יום מדלגות על מי שכבר נשלח.

## אזור זמן
החישוב לפי `Asia/Jerusalem` (תאריך ושעה), לא UTC, כדי ששעון קיץ/חורף לא יזיז את שעת ההתראה.

## סודות
- `CRON_SECRET` - גם כ-Edge Function Secret ב-Supabase וגם כ-GitHub Secret (אותו ערך). מאמת שרק ה-cron שלנו מפעיל את הפונקציה.
- `TELEGRAM_BOT_TOKEN` - כבר מוגדר (משותף לכל הפונקציות).

## פריסה
1. בדאשבורד: Edge Functions -> create `send-morning`, הדבק את הקוד, **Verify JWT OFF**, Deploy.
2. הוסף Secret `CRON_SECRET`.
3. ה-workflow כבר במאגר; ירוץ לפי הלו"ז (ואפשר Run workflow ידנית לבדיקה).

## מגבלות
- מכוסה כרגע רק ערוץ טלגרם. Web Push יתווסף בנפרד.
- "פריט נוסף אחרי שכבר נשלחה הודעת הבוקר" לא יישלח שוב באותו יום (נשלח פעם אחת ביום).
